# CER 정답원문 작성 안내 (H1 / 논문 표6)

이 폴더의 빈 `.txt` 20개를 **사람이 본 정확한 전체 텍스트**로 채우면 됩니다.
나중에 `CER = (치환+삭제+삽입) / 정답글자수` 로 시스템 파싱결과(`run_<ts>/parsed/`)와 자동 비교합니다.

## 작성 규칙
- 해당 원본 문서를 열어 **실제로 있는 텍스트를 그대로 타이핑** (UTF-8 저장).
- 줄바꿈·공백은 신경 안 써도 됨 → CER 계산 전 정규화(공백 합치기) 적용.
- 숫자·금액·이름은 **있는 그대로** (예: `9,845,000`, `김 인 성`).
- 페이지번호·로고·머리말 같은 잡요소는 빼도 됨.
- **표**: 셀 내용을 위→아래, 좌→우 순서로 풀어서.
- **음성**: 들으면서 말한 내용 그대로 받아쓰기(화자 구분 불필요).
- **이미지에 글자가 없으면**(순수 사진) → 파일 비워두고 파일명 끝에 메모 없이 그대로 두세요. (CER 계산에서 자동 제외)
- **txt 2건**은 거의 대조군 — `data/eval_set/`의 동일 txt 내용을 그대로 붙여넣으면 됨(파싱이 trivial).

## 표·기호 처리 (중요)
- **표(병합 셀 포함)**: 격자/칸선을 흉내 내지 말고 **읽는 순서(위→아래, 좌→우)로 셀 안 텍스트만** 적기.
  병합된 큰 칸(예: "시장성 없는 자산의 취득 또는 처분")은 **한 번만** 쓰고 하위 라벨(증권/실물/…)과 내용을 이어 쓴다.
- **표 기호(`|`, 칸선)는 넣지 않는다** — CER 계산 전 양쪽에서 마크다운/표기호·공백을 제거해 공정 비교한다.
- **중간점 `·`(가운뎃점)**: 문서에 있으면 **그대로 입력**(예: 부동산·특별자산펀드). (논문 본문에서 ·를 뺀 것은 글쓰기 스타일일 뿐, 정답원문은 문서 충실 재현)
- **숫자·금액·날짜·영문 고유명사**는 보이는 그대로 정확히(예: `$8,268,427.20`, `10.51%`, `2020년 12월 15일`, `MAPS 2157 STACIES WAY IN LLC`).
